## Recovery result

The registry-installed CLI operated on a fresh Postgres 16 database initialized at application release `0.1.4`. It planned and preflighted `0.1.4` to `0.2.1`, created and verified the rollback backup, applied the packaged `0.2.1` hook, verified the release marker and migration ledger, and completed an isolated rollback drill. The active-restore path then created and verified a safety backup, restored the rollback target to staging, swapped the active database back to `0.1.4`, retained the pre-rollback database, and verified the restored live result.

The retained [active rollback receipt](./v0.2.1-published-rollback-receipt.json) records `mode=active_database_restore`, `fromVersion=0.2.1`, `toVersion=0.1.4`, both verified backup identities and hashes, the preserved pre-rollback database and OID, `postSafetyWritesPreserved=true`, and `postRestoreVerified=true`.
